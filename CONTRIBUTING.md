# Contributing to Chester

Thank you for your interest in improving Chester! This skill helps AI coding agents treat legacy code with the care it deserves, and contributions that sharpen its guidance are welcome.

## How to contribute

### Reporting issues

If Chester's guidance led to a bad outcome — a fence it should have caught, a false alarm that wasted effort, or unclear instructions — [open an issue](../../issues/new) describing:

- The situation (what kind of code, what task)
- What Chester directed the agent to do
- What should have happened instead

### Suggesting improvements

Ideas for new fence patterns, better archaeology techniques, or workflow refinements are welcome as issues or pull requests.

### Submitting changes

1. Fork the repository
2. Create a branch (`feat/your-improvement` or `fix/your-fix`)
3. Edit `SKILL.md` (the source of truth) and, if needed, rebuild `chester.skill`
4. Write a clear commit message explaining *why* the change improves the skill
5. Open a pull request

### What makes a good contribution

- **Concrete over abstract.** Real examples of fence patterns beat theoretical descriptions.
- **Battle-tested.** If you've seen a fence pattern cause real problems when removed, that experience is especially valuable.
- **Precise.** Small, focused changes are easier to review than sweeping rewrites.
